# Commit message style (user preference)

Keep commit messages SHORT: 1-2 lines, subject only. No body paragraphs unless asked.
Do NOT add `Co-Authored-By` / co-authoring trailers.

Example accepted: `Feature 3. US1: concrete evidence for every ERROR/WARN`

**Why:** user rejected a long multi-paragraph commit with co-author trailer; wants terse one-liners.
